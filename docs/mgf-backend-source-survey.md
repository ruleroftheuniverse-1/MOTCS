# MgF backend source survey

Survey date: 2026-06-24

Implementation status: the first validation adapter now lives in
`src/mgf_mot/mgf_backend.py`, with source-tagged data in
`src/mgf_mot/spectroscopy.py` and a report script at
`scripts/validate_mgf_hamiltonian.py`. It validates the sourced ground
Hamiltonian and the 12+4 angular structure; it is not a complete MgF
Hamiltonian or force backend.

## Decision

We found a **credible, reusable partial backend**, but no drop-in, provenance-complete
MgF implementation of the Rodriguez static force maps.

The strongest path is **A: adapt the official `pylcp` molecular backend**:

- `pylcp.hamiltonians.XFmolecules.Xstate` constructs an alkaline-earth-fluoride
  `X 2Sigma+` Hamiltonian and magnetic-moment operators;
- `Astate` constructs an `A 2Pi_1/2` Hamiltonian and magnetic-moment operators;
- `dipoleXandAstates` constructs relative spherical transition-dipole matrices;
- `pylcp.rateeq` supplies the force solver; and
- the official CaF example demonstrates this complete workflow.

With `N=1`, `I=1/2`, `J'=1/2`, and one excited parity, the installed backend
produces exactly 12 ground states, 4 excited states, and a dipole tensor of shape
`(3, 12, 4)`. This was checked locally without assigning any MgF constants.

This is not yet sufficient to implement MgF honestly. The MgF parameters must be
entered from the spectroscopy sources below, their conventions must be mapped to
the backend, the excited-state hyperfine model must be checked for the independent
`d` term used by Doppelbauer et al., and the Zeeman and dipole matrices must be
validated. The historical MgF defaults found in old code are explicitly **not** an
acceptable parameter source.

## Search scope and limitations

The search covered:

- this repository and its notes;
- installed `pylcp 1.0.2` source under
  `.venv/Lib/site-packages/pylcp`;
- the official `JQIamo/pylcp` repository, history, tests, and notebooks;
- locally held PDFs in `MOT sim papers/`;
- GitHub repository and code searches for the requested terms;
- Crossref records and arXiv primary records/PDFs.

Search terms included MgF, molecular MOT, Rodriguez, Norrgard, Eckel, Barker,
chirped MOT, type-II MOT, 12 ground/4 excited states, hyperfine constants,
spin-rotation, dipolar hyperfine, transition dipole matrices, and rate-equation
force model.

GitHub repository search returned no repository for the exact MgF/Rodriguez query
sets. GitHub code search found the official generic molecular backend and CaF
example, but no public Rodriguez MgF parameter file or force-map notebook. The
general web-search endpoint returned HTTP 403, so primary-source discovery used
GitHub directly, Crossref, arXiv, and the local papers instead.

## Backend and code candidates

### 1. Official `pylcp` molecular Hamiltonian utilities — best backend candidate

- **Path:** `.venv/Lib/site-packages/pylcp/hamiltonians/XFmolecules.py`
- **URL:** [JQIamo/pylcp `XFmolecules.py`](https://github.com/JQIamo/pylcp/blob/v1.0.2/pylcp/hamiltonians/XFmolecules.py)
- **Contains:** generic Hund's-case basis builders for `X 2Sigma+` and
  `A 2Pi_1/2`, field-free Hamiltonians, spherical magnetic-moment operators,
  and relative `X`-to-`A` dipole matrices built from Wigner symbols.
- **Hamiltonian constants included?** Parameter slots and CODATA magnetic
  constants, but deliberately no MgF spectroscopy values in current releases.
- **Transition/coupling matrices?** Yes. `dipoleXandAstates` returns relative
  `q=-1,0,+1` dipole matrices.
- **Force-map/MOT code?** Indirectly: its output is accepted by the official
  `pylcp.hamiltonian` and `pylcp.rateeq` classes.
- **Rodriguez compatibility:** High structural compatibility. The generated
  12/4 basis sizes and 3x12x4 coupling tensor match the stated Rodriguez
  manifold.
- **Confidence:** High for reuse as a framework; medium for unmodified MgF
  fidelity.
- **Remaining issues:** map every spectroscopy convention; add or validate the
  excited hyperfine `d` interaction (the public `Astate` API has `a`, `b`, and
  `c`, but no explicit `d` argument or independently adjustable `d` operator);
  verify the full MgF Zeeman model;
  and address a locally observed `pylcp 1.0.2` compatibility bug where
  `dipoleXandAstates(..., UX=<numpy array>)` evaluates `UX == []` and raises a
  broadcasting `ValueError`.

### 2. Official CaF MOT example — best workflow template

- **URL:** [official `20_CaF_MOT.ipynb`](https://github.com/JQIamo/pylcp/blob/v1.0.2/docs/examples/MOTs/20_CaF_MOT.ipynb)
- **Contains:** construction of `X` and `A` Hamiltonians, dipole matrices,
  `pylcp.hamiltonian`, multicomponent six-beam MOT fields, `rateeq`, position
  and velocity force profiles, and a two-color MOT.
- **Hamiltonian constants included?** Yes, but for CaF only.
- **Transition/coupling matrices?** Yes, generated by `dipoleXandAstates`.
- **Force-map/MOT code?** Yes.
- **Rodriguez compatibility:** High as an architectural template; its species,
  beam geometry, frequencies, and parameters are not transferable.
- **Confidence:** High.
- **Remaining issues:** replace only with sourced MgF inputs, rotate the beam
  geometry to Rodriguez x-prime/y-prime axes, and reproduce Rodriguez component
  grouping and normalization.

### 3. Official `pylcp` rate-equation engine and package paper

- **Paths:** `.venv/Lib/site-packages/pylcp/rateeq.py` and
  `MOT sim papers/Eckel 2020.pdf`
- **URLs:** [official repository](https://github.com/JQIamo/pylcp),
  [package paper DOI](https://doi.org/10.1016/j.cpc.2021.108166)
- **Contains:** generic Hamiltonian blocks, laser/magnetic fields,
  rate-equation and optical-Bloch-equation solvers, forces, and trajectories.
- **Hamiltonian constants included?** No MgF constants.
- **Transition/coupling matrices?** Consumes them; generic atomic and molecular
  helpers generate them.
- **Force-map/MOT code?** Yes.
- **Rodriguez compatibility:** High; Rodriguez cites this package as ref. 38.
- **Confidence:** High.
- **Remaining issues:** all molecule-specific inputs remain external.

### 4. Historical MgF defaults in old `XFmolecules.py` — lead only, do not reuse

- **Historical URL:** [official commit removing the defaults](https://github.com/JQIamo/pylcp/commit/c092629)
- **Surviving fork URL:** [`awesomelw/pylcp-nist`](https://github.com/awesomelw/pylcp-nist)
- **Contains:** old defaults labeled `14MgF` in `Xstate` (almost certainly a
  metadata typo and itself a warning sign), including `B`, `gamma`, `b`, and
  `c`; an old simplified `Astate`; generic dipole matrices.
- **Hamiltonian constants included?** Values are present, but without adequate
  citations or uncertainty/provenance in the file.
- **Transition/coupling matrices?** Yes, generic angular factors.
- **Force-map/MOT code?** The fork contains generic and CaF examples, not a
  Rodriguez MgF force map.
- **Rodriguez compatibility:** Superficially close, but not trustworthy as-is.
  The old `Astate` docstring explicitly said it was taking guesses for
  gyromagnetic ratios. Official `pylcp` removed the MgF defaults in 2020 and
  changed the functions to require explicit parameters.
- **Confidence:** Low as a constants source; medium as historical evidence that
  these utilities were originally developed with MgF in mind.
- **Remaining issues:** provenance, a ground hyperfine-convention mismatch
  (`b` versus `b_F`), suspect units/default `muB`, simplified excited
  Hamiltonian, and guessed magnetic parameters.

### 5. `PylcpGPU`

- **URL:** [EugeneZhu0505/PylcpGPU](https://github.com/EugeneZhu0505/PylcpGPU)
- **Contains:** a GPU-oriented copy/derivative of generic `pylcp`, including
  `XFmolecules.py`.
- **Hamiltonian constants included?** No credible MgF parameter set.
- **Transition/coupling matrices?** Generic copy only.
- **Force-map/MOT code?** Generic examples.
- **Rodriguez compatibility:** No advantage over official `pylcp` for this task.
- **Confidence:** Low as an MgF candidate.
- **Remaining issues:** provenance and divergence from upstream.

### 6. `pyLCP-Example-Code`

- **URL:** [lillevvink/pyLCP-Example-Code](https://github.com/lillevvink/pyLCP-Example-Code)
- **Contains:** general atomic/isotope examples and saved results.
- **Hamiltonian constants included?** Atomic examples only.
- **Transition/coupling matrices?** Atomic examples only.
- **Force-map/MOT code?** General `pylcp` demonstrations.
- **Rodriguez compatibility:** None beyond generic API usage.
- **Confidence:** Low as an MgF candidate.

## MgF papers and validation candidates

### 7. Rodriguez et al., target calculation

- **Path:** `MOT sim papers/Chirped_MOT_Theory.pdf.pdf`
- **URL:** [Phys. Rev. A 108, 033105](https://doi.org/10.1103/PhysRevA.108.033105),
  [arXiv:2305.04879](https://arxiv.org/abs/2305.04879)
- **Contains:** the target 16-state model description, level spacings,
  effective level g-factors, linewidth, laser components, beam geometry,
  static/chirped force maps, and simulation parameters.
- **Hamiltonian constants included?** No. It points to refs. 18 and 34 for
  spectroscopic parameters and uses an explicitly representative excited
  `g=0.001` because its sign and precise value were unknown.
- **Transition/coupling matrices?** No tables or source matrices.
- **Force-map/MOT code?** Results and method, but no public code found.
- **Compatibility:** This is the reference target.
- **Confidence:** High for target conventions and observables; insufficient as
  a standalone Hamiltonian source.
- **Remaining issues:** source-level Hamiltonian assembly, parameter mapping,
  transition matrices, and exact excited-state energy details.

### 8. Xu et al. 2019 MgF multilevel rate-equation MOT

- **Path:** `MOT sim papers/Xu 2019.pdf`
- **URL:** [Phys. Rev. A 99, 033408](https://doi.org/10.1103/PhysRevA.99.033408),
  [arXiv:1903.00609](https://arxiv.org/abs/1903.00609)
- **Contains:** a 3D MgF rate-equation model, three-, four-, and 3+1-frequency
  MOT studies, ground hyperfine energy offsets, effective g-factors, force
  profiles, and rate equations.
- **Hamiltonian constants included?** No full Hamiltonian constant set; it uses
  level energies/effective g-factors.
- **Transition/coupling matrices?** Relative transition/branching strengths are
  used, but no reusable matrix file or public code was found.
- **Force-map/MOT code?** Equations and published plots, not source code.
- **Rodriguez compatibility:** High as an independent qualitative and numerical
  validation source. Rodriguez explicitly builds on its MgF configurations.
- **Confidence:** High as a validation source, medium as a source of effective
  level inputs.
- **Remaining issues:** no machine-readable backend, differing linewidth and
  gradient choices, and no complete Zeeman/dipole operator construction.

### 9. Doppelbauer et al. 2022 MgF spectroscopy — primary constants source

- **URL:** [J. Chem. Phys. 156, 134301](https://doi.org/10.1063/5.0081902),
  [arXiv:2112.06555](https://arxiv.org/abs/2112.06555)
- **Rodriguez bibliography:** ref. 18.
- **Contains:** hyperfine-resolved `A 2Pi <- X 2Sigma+` spectroscopy; reproduced
  ground `B`, `D`, `gamma`, `b_F`, and `c`; fitted excited combinations
  `A+gamma`, `p+2q`, `a`, `b_F+2c/3`, and `d`; line assignments and effective
  Hamiltonian details.
- **Hamiltonian constants included?** Yes, with uncertainties and explicit
  convention notes. Some parameters are only determined as correlated linear
  combinations.
- **Transition/coupling matrices?** No machine-readable matrices; it discusses
  weak branching caused by hyperfine mixing.
- **Force-map/MOT code?** No.
- **Rodriguez compatibility:** Directly cited and highly relevant.
- **Confidence:** High.
- **Remaining issues:** translate its Frosch-Foley conventions into the exact
  `pylcp` API; decide how to handle correlated rather than separately measured
  constants; and verify that `pylcp.Astate` implements the independent `d` term.

### 10. Anderson, Allen, and Ziurys 1994 ground-state spectroscopy

- **URL:** [J. Chem. Phys. 100, 824](https://doi.org/10.1063/1.466565)
- **Rodriguez bibliography:** ref. 34.
- **Contains:** pure rotational spectroscopy of `X 2Sigma+`, including
  rotational/fine constants and fluorine hyperfine parameters for `24MgF`.
- **Hamiltonian constants included?** Yes for the ground state.
- **Transition/coupling matrices?** No.
- **Force-map/MOT code?** No.
- **Rodriguez compatibility:** Directly cited; Doppelbauer reproduces the
  relevant `v=0` constants and explains the `b` versus `b_F` convention.
- **Confidence:** High.
- **Remaining issues:** obtain/record exact table, units, uncertainties, and
  convention mapping before code.

### 11. Xu et al. 2019 high-resolution `A-X` spectroscopy

- **URL:** [J. Chem. Phys. 150, 084302](https://doi.org/10.1063/1.5083898)
- **Rodriguez bibliography:** ref. 33.
- **Contains:** identification of the laser-cooling transition and the ground
  hyperfine pattern; establishes the normal ordering of the `A 2Pi` state.
- **Hamiltonian constants included?** Spectroscopic fits, but Doppelbauer is the
  more direct high-resolution source used by Rodriguez for the model parameters.
- **Transition/coupling matrices?** No reusable matrices found.
- **Force-map/MOT code?** No.
- **Rodriguez compatibility:** Directly cited for the transition.
- **Confidence:** High for transition identification, medium for final model
  constants.
- **Remaining issues:** reconcile any fit differences with Doppelbauer and the
  target paper.

### 12. Pilgram et al. 2024 — strongest independent `pylcp` validation

- **Path:** `MOT sim papers/Pilgram 2024.pdf`
- **URL:** [Phys. Rev. A 110, 023110](https://doi.org/10.1103/PhysRevA.110.023110),
  [arXiv:2403.11785](https://arxiv.org/abs/2403.11785)
- **Contains:** measured MgF laser-cooling transition frequencies, explicit
  effective Hamiltonians, and the statement that the molecular Hamiltonians
  and transition-dipole matrix elements were calculated with `pylcp`. It uses
  Anderson ground parameters and Doppelbauer `A`-state hyperfine parameters.
- **Hamiltonian constants included?** Yes for its fitted `B` state; references
  the needed `X` and `A` sources for the present transition.
- **Transition/coupling matrices?** Generated in `pylcp`, but not published as
  a matrix file.
- **Force-map/MOT code?** No source code or static force map in the paper.
- **Rodriguez compatibility:** Very high methodological compatibility and a
  strong validation target for zero-field line positions/strengths.
- **Confidence:** High.
- **Remaining issues:** no released MgF builder code was found.

### 13. Norrgard et al. 2023 decay and branching measurements

- **Path:** `MOT sim papers/Norrgard 2023.pdf`
- **URL:** [Phys. Rev. A 108, 032809](https://doi.org/10.1103/PhysRevA.108.032809),
  [arXiv:2303.16276](https://arxiv.org/abs/2303.16276)
- **Contains:** measured `A 2Pi_1/2` decay rate and vibrational branching
  fractions, calculated vibronic transition dipole moments, and a statement
  that realistic MgF MOT simulations used `pylcp` and are detailed in the
  related Rodriguez work.
- **Hamiltonian constants included?** Not the 16-state hyperfine/Zeeman set.
- **Transition/coupling matrices?** Vibronic TDM information, not the complete
  Zeeman-resolved angular matrix needed here.
- **Force-map/MOT code?** No public code found.
- **Rodriguez compatibility:** High for linewidth and decay assumptions.
- **Confidence:** High.
- **Remaining issues:** combine its total decay/vibrational data with angular
  branching from a validated dipole tensor.

### 14. General force-model validation papers

- **Paths:** `MOT sim papers/Devlin 2017.pdf`,
  `MOT sim papers/Tarbutt 2025.pdf`, and the CaF discussions in the official
  `pylcp` notebook.
- **Contains:** type-II molecular MOT force behavior, polarization/g-factor
  rules, dark states, and multilevel rate/OBE comparisons.
- **Hamiltonian constants included?** Species-specific examples, not MgF.
- **Transition/coupling matrices?** Theoretical angular models, no MgF matrix.
- **Force-map/MOT code?** Published calculations; no MgF backend found.
- **Rodriguez compatibility:** Useful qualitative cross-checks only.
- **Confidence:** High for general theory.
- **Remaining issues:** none of these substitutes for MgF spectroscopy.

### 15. Later chirped-MOT and MgF optical-cycling work

- **Path:** `MOT sim papers/Paladugu 2025.pdf`
- **URL:** [Paladugu et al., chirped MOT of molybdenum](https://doi.org/10.1103/49lr-6slw),
  [MgF optical cycling, arXiv:2506.23507](https://arxiv.org/abs/2506.23507),
  [MgF `v=2 -> v'=1` spectroscopy, arXiv:2603.29309](https://arxiv.org/abs/2603.29309)
- **Contains:** experimental chirped-MOT validation in Mo and later MgF
  optical-cycling/spectroscopy results.
- **Hamiltonian constants included?** The 2026 paper reports constants for a
  different vibrational transition; it is not the source for the Rodriguez
  `v=0 -> v'=0` 16-state model.
- **Transition/coupling matrices?** No public Rodriguez matrix/backend found.
- **Force-map/MOT code?** No reusable MgF force-map code found.
- **Rodriguez compatibility:** Context and future validation, not a backend.
- **Confidence:** Medium for relevance to this narrowly scoped replication.

## Bibliography mapping for the Rodriguez Hamiltonian

The relevant sentence in Rodriguez states that the model transition is computed
using refs. **18 and 33**, and then states that the relevant spectroscopic MgF
parameters are in refs. **18 and 34**.

| Rodriguez ref. | Source | Role in a faithful implementation |
|---|---|---|
| 18 | Doppelbauer et al., JCP 156, 134301 (2022) | Primary `A`-state hyperfine, rotation, and lambda-doubling source; also reproduces ground constants and conventions. |
| 33 | S. Xu et al., JCP 150, 084302 (2019) | Laser-cooling transition identification, state ordering, and observed ground hyperfine structure. |
| 34 | Anderson, Allen, and Ziurys, JCP 100, 824 (1994) | Primary `X(v=0)` rotation, spin-rotation, and fluorine hyperfine constants. |
| 35 | Brown et al., Molecular Physics 36, 553 (1978) | Effective `2Pi` Zeeman terms referenced by Rodriguez. |
| 36 | Brown and Carrington, *Rotational Spectroscopy of Diatomic Molecules* (2003) | Hamiltonian and angular-momentum convention reference used by `pylcp.XFmolecules`. |
| 38 | Eckel et al., CPC 270, 108166 (2022) | `pylcp` solver/backend reference. |

Refs. 19 (Norrgard et al. 2023) and 33/34 also support linewidth, branching,
transition frequency, and validation, but do not supply a ready-to-run MgF file.

## Bottom line

- **Reusable backend:** yes, official generic `pylcp.XFmolecules` plus
  `pylcp.rateeq`.
- **Reusable MgF parameter module:** no credible current one found.
- **Reusable MgF force-map code:** none found publicly.
- **Partial MgF source:** yes: an old unverified parameter default, published
  rate-equation results, and strong spectroscopy sources.
- **Next step:** **A — adapt the official `pylcp` molecular backend**, with all
  MgF parameters transcribed from refs. 18/34 (and supporting refs. 33/19), an
  explicit convention ledger, a reviewed treatment of the excited `d` term and
  Zeeman interactions, and validation against published level positions and
  dipole-strength sum rules before any force map is trusted.

## Validation-layer result

The implemented adapter currently establishes the following without adding a
force calculation:

- official `pylcp.Xstate` plus Anderson/Doppelbauer ground constants produces
  12 eigenstates in four levels ordered `lower F=1`, `F=0`, `upper F=1`, `F=2`;
- calculated consecutive spacings are 109.732, 120.327, and 9.268 MHz, matching
  the rounded Rodriguez Fig. 1 targets 109, 120, and 9 MHz;
- official `pylcp.Astate` enumerates the required four-state positive-parity
  `J'=1/2`, `F'=0,1` basis, but its returned zero-default Hamiltonian is
  discarded and never represented as MgF data;
- official angular coupling code, with the documented `UX` adapter, produces a
  tensor of shape `(3, 12, 4)`; and
- complete `pylcp.hamiltonian` construction raises a capability error listing
  the independent `d` term and unresolved parameter combinations.

This changes the survey classification from “candidate only” to **validated
partial backend**. It does not change the conclusion that no provenance-complete
drop-in MgF implementation was found.

### Factory blocker follow-up

The exact full factory still fails intentionally: the public `pylcp.Astate`
interface has no independent Doppelbauer `d` input, the excited hyperfine and
lambda-doubling constants are not all separately mapped to backend arguments,
and the excited Zeeman constants remain unresolved.

An explicit provisional path,
`ApproximationMode.COLLAPSED_PYLCP_ASTATE`, can create a pylcp-compatible
two-block object only when requested. It reports the omitted independent `d`
term, collapsed excited hyperfine combination, zeroed unresolved Zeeman terms,
and confirms that no undocumented pylcp defaults were used. This approximation
is not enabled by default and is not accepted as force-ready.
